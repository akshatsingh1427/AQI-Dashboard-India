India AQI Monitoring Dataset

File: AQI.csv — compiled from India's national air quality monitoring network. Covers eight core attributes: country, state, city, last_update (timestamp), pollutant_id, pollutant_min, pollutant_max, and pollutant_avg.

Columnar Structure
Eight columns record geographic identifiers (country, state, city), a datetime stamp (last_update), the pollutant identifier (pollutant_id), and three pollutant readings: minimum, maximum, and average concentration.
Temporal Scope
Data spans four complete years — 2022 through 2025 — providing sufficient longitudinal depth for trend analysis, seasonal decomposition, quarterly breakdown, and future-year prediction modelling.
Geographic Coverage
All Indian states and union territories are represented, enabling both state-level granularity and 7-region aggregation across North, South, East, West, Central, Northeast, and Islands.

Principles of Formal Data Analysis (FDA)
Foundational analytical principles applied throughout this project
Formal Data Analysis is a structured, reproducible approach to understanding data through systematic preprocessing, exploration, statistical inference, and communication. The following core principles guided all decisions in this project.
🧹
Data Integrity
All missing values are handled explicitly before any analysis. No imputation is performed silently — null regions are either dropped or filled with zero where appropriate (e.g., heatmap pivots).
🔍
Exploratory First
Exploratory Data Analysis (EDA) precedes modelling. Distributions, central tendencies, outliers and group structures are examined visually before any predictive algorithm is trained.
📐
Feature Engineering
Raw datetime is decomposed into year, month, month_name, and quarter features. A new categorical variable — region — is derived from state names using a domain-knowledge mapping.
📊
Aggregation Discipline
All group comparisons use mean aggregation on pollutant_avg. Pivot tables, groupby chains, and merge operations are applied consistently to avoid double-counting or misaligned indices.
🔬
Multi-level Analysis
Analysis is conducted at three geographic levels simultaneously — national, regional, and state — allowing findings at each level to corroborate or contextualise the others.
📈
Predictive Validation
Predictions are computed only when at least two historical data points exist per entity (state/region). Negative predicted values are clamped to zero to preserve physical plausibility.
🎯
Reproducibility
All transformations, groupings, and model fits are contained in a single deterministic Python script. Running the script on the same input always produces the identical output.
📣
Communicability
Every analytical finding is translated into an interactive Plotly visualisation embedded in a structured HTML dashboard, ensuring results are accessible to non-technical stakeholders.

Data Preprocessing & EDA Steps
Sequential pipeline from raw CSV to analysis-ready dataframe
01
Data Ingestion
The dataset is loaded from AQI.csv using pandas.read_csv(). The resulting dataframe contains eight columns covering geography, timestamps, pollutant identifiers, and concentration readings (min, max, avg).
pandas
02
Datetime Parsing
The last_update column is converted from string to datetime using pd.to_datetime(). Four derived temporal features are then extracted: year, month, month_name, and quarter, enabling temporal grouping at multiple granularities.
Feature Engineering
03
Null Value Handling
Null values arise in two places: (a) the monthly pivot table, where year-month combinations with no readings produce NaN — these are filled with 0 using fillna(0); and (b) the region column for unmapped states — these are filtered out with pd.notna() before regional aggregation and prediction loops.
Data Quality
04
Region Feature Engineering
A domain-knowledge dictionary maps each of India's 36 states and UTs to one of seven geographic regions: North, South, East, West, Central, Northeast, and Islands. This new region column enables an intermediate geographic level of analysis between national and state.
Domain Knowledge Feature Engineering
05
Pollutant Level Categorisation
A custom function categorize_pollutant_level() applies rule-based classification to pollutant_avg: values above 100 are labelled High, values between 50 and 100 are Medium, and values below 50 are Low. This categorical variable is appended to the state-pollutant and region-pollutant analysis dataframes.
Binning Rule-based Classification
06
Aggregation & Pivot Operations
Multiple group-level aggregations are computed: national yearly trend, monthly averages by year (pivoted to a 12×4 matrix for the heatmap), state-level means, region-level means, state-pollutant cross-tabulations, region-pollutant cross-tabulations, and city count per state. All aggregations use mean as the central statistic on pollutant_avg.
GroupBy Pivot Tables
07
State-Pollutant Pivot Matrix
A 2D pivot table is created with states as rows and pollutant types as columns, with pollutant_avg as values and zero-fill for missing combinations. This matrix is used directly to render the State × Pollutant concentration heatmap.
Matrix Construction
08
Per-Region Top & Bottom State Extraction
For each of the seven regions, the three most polluted and three cleanest states are extracted using nlargest() and nsmallest() on state-level averages within the region. These are concatenated into unified dataframes for faceted visualisation.
Stratified Ranking
09
Dominant Pollutant Identification
For each state and each region, the pollutant with the highest average concentration is identified using idxmax() applied to grouped data. This supports the dominant-pollutant bar chart and the summary table in the Pollutants tab.
Argmax Selection
10
City-Count Merge for Scatter Analysis
The count of unique cities per state (nunique() on the city column) is computed separately and merged with state-level pollution averages and region labels. The merged dataframe drives the scatter plot comparing urbanisation proxy (city count) against average AQI.
Merge / Join Proxy Variable
4
Predictive Models Used
Machine learning algorithms employed to forecast AQI for 2026
Polynomial Regression (Degree 2) — Primary Model
scikit-learn · PolynomialFeatures + LinearRegression

For each state and region individually, year values (2022–2025) are used as the sole feature. PolynomialFeatures(degree=2) expands the single feature into a quadratic basis [1, year, year²], which is then fitted via ordinary least squares using LinearRegression. The trained model is used to predict the pollutant average for the year 2026.

Polynomial regression is preferred over simple linear regression here because air quality trends can exhibit non-linear acceleration or deceleration — particularly in states with rapid industrialisation or effective policy interventions during the observed window.

Linear Regression — Fallback Model
scikit-learn · LinearRegression
When the polynomial model raises an exception — for example, when only two data points exist for a state — the pipeline falls back to a simple LinearRegression fitted directly on year values. This ensures every state and region with at least two historical observations receives a 2026 forecast rather than being omitted from the analysis.

Trend Classification: Each predicted 2026 value is compared against the 2022–2025 historical mean. If the prediction exceeds the historical mean, the state/region is labelled Increasing. If below, it is labelled Decreasing. If equal, it is labelled Stable. The percentage change is also computed and displayed in the predictions table.

Libraries & Technologies
Python packages and front-end technologies used throughout the project
pandas
Primary data manipulation library. Used for CSV ingestion, datetime parsing, groupby aggregations, pivot tables, merge operations, null handling, and dataframe filtering throughout the entire preprocessing and EDA pipeline.
NumPy
Used for numerical operations including array reshaping for scikit-learn input (reshape(-1,1)), rounding heatmap annotation values, and computing waterfall deltas.
Plotly Express
High-level Plotly interface used to produce bar charts, line charts, scatter plots, area charts, box plots, violin plots, pie/donut charts, sunburst charts, treemaps, and faceted plots with minimal code.
Plotly Graph Objects
Low-level Plotly interface used for custom multi-trace figures: heatmaps with annotations, waterfall charts, funnel charts, radar/spider charts, and the national prediction line chart with dual traces.
scikit-learn
Machine learning library supplying LinearRegression for ordinary least squares fitting and PolynomialFeatures for quadratic basis expansion. Used to train and predict AQI values for 2026 at state, regional, and national levels.
HTML5 / CSS3 / JavaScript
The dashboard output is a self-contained HTML file with embedded Plotly JS for interactivity, custom CSS for the dark theme and square-box tab navigation, and vanilla JS for tab switching, scroll progress, particle animation, and counter animation.
